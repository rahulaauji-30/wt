Constraints

In Oracle Database, constraints are rules defined on a table to enforce data integrity. Constraints help ensure that the data stored in the database is accurate, reliable, and consistent. Here are some commonly used constraints in Oracle DBMS.

  1. PRIMARY KEY Constraint
    • Ensures that a column or a combination of columns uniquely identifies each row in a table. 
                              
                            CREATE TABLE employees (
                            employee_id NUMBER PRIMARY KEY,
                            first_name VARCHAR2(50),
                            last_name VARCHAR2(50),
                            hire_date DATE,
                            salary NUMBER
                            );
  2. FOREIGN KEY Constraint
    • Establishes a link between two tables by enforcing referential integrity. The foreign key in one table must match the primary key in another table. 
                              
                            CREATE TABLE orders (
                            order_id NUMBER PRIMARY KEY,
                            employee_id NUMBER,
                            order_date DATE,
                            FOREIGN KEY (employee_id) REFERENCES employees(employee_id)
                        );
  3. UNIQUE Constraint
    • Ensures that all values in a column or a combination of columns are unique across the table. 
                              
                            CREATE TABLE departments (
                            department_id NUMBER PRIMARY KEY,
                            department_name VARCHAR2(50) UNIQUE,
                            location VARCHAR2(100)
                            );
  4. CHECK Constraint
    • Restricts the range of valid values for a column. 
                              
                            CREATE TABLE employees (
                            employee_id NUMBER PRIMARY KEY,
                            first_name VARCHAR2(50),
                            last_name VARCHAR2(50),
                            hire_date DATE,
                            salary NUMBER CHECK (salary > 0)
                            );
  5. NOT NULL Constraint
    • Ensures that a column does not contain any NULL values. 
                              
                            CREATE TABLE students (
                            student_id NUMBER PRIMARY KEY,
                            student_name VARCHAR2(50) NOT NULL,
                            enrollment_date DATE
                            );
  6. DEFAULT Constraint
    • Specifies a default value for a column if no value is provided during an insert operation. 
                              
                            CREATE TABLE employees (
                            employee_id NUMBER PRIMARY KEY,
                            first_name VARCHAR2(50),
                            last_name VARCHAR2(50),
                            hire_date DATE DEFAULT SYSDATE,
                            salary NUMBER DEFAULT 50000
                            );
  7. INDEX Constraint
    • Creates an index on one or more columns to enhance query performance. 
                              
                            CREATE INDEX idx_last_name ON employees(last_name);
  8. ENABLE/DISABLE Constraint Enables or disables a constraint. This is useful when you want to temporarily suspend the enforcement of a constraint. 
                    
                    ALTER TABLE employees
                    DISABLE CONSTRAINT salary_check;
  9. COMPOSITE Constraints
    • You can combine multiple constraints on a single column or across multiple columns.
      •                     
                        CREATE TABLE orders (
                        order_id NUMBER PRIMARY KEY,
                        product_id NUMBER,
                        quantity NUMBER CHECK (quantity > 0),
                        FOREIGN KEY (product_id) REFERENCES products(product_id)
                        );
  10. DEFERRABLE Constraints
    • Allows deferring checking constraints until the end of a transaction.
      •                     
                        CREATE TABLE transactions (
                        transaction_id NUMBER PRIMARY KEY,
                        account_id NUMBER,
                        amount NUMBER,
                        CHECK (amount > 0) DEFERRABLE INITIALLY DEFERRED,
                        FOREIGN KEY (account_id) REFERENCES accounts(account_id)
                        );
