Indexes

Indexes in Oracle Database are database objects that provide a fast and efficient way to retrieve data from tables. They enhance the speed of data retrieval operations but can also impact the performance of data modification operations (insert, update, delete). Here's additional information about indexes in Oracle DBMS.

  1. Creating an Index

    • Use the CREATE INDEX statement to create an index on one or more columns of a table. Here's an example:

                              
                            CREATE INDEX idx_last_name ON employees(last_name);
  2. Types of Indexes

    • B-Tree Indexes: The most common type of index in Oracle. It organizes data in a balanced tree structure.

                              
                            CREATE INDEX idx_salary ON employees(salary);

    • Bitmap Indexes: Suitable for columns with low cardinality (few distinct values).

                              
                            CREATE INDEX idx_salary ON employees(salary);

    • Function-Based Indexes: Created based on a function applied to the columns.

                              
                            CREATE INDEX idx_upper_last_name ON employees(UPPER(last_name));

    • Clustered Indexes: In Oracle, the equivalent is a Cluster, which is a group of tables physically stored together based on the clustering key.

                              
                            CREATE CLUSTER employees_cluster (employee_id NUMBER)
                            SIZE 8192;
  3. Unique Index
    • Ensures that all values in the indexed column(s) are unique. 
                              
                            CREATE UNIQUE INDEX idx_email ON employees(email);
  4. Descending Index
    • Specifies that the index should be in descending order. Useful for sorting queries in descending order. 
                              
                            CREATE INDEX idx_hire_date_desc ON employees(hire_date DESC);
  5. Function-Based Index
    • Creates an index based on the result of a function or expression. 
                              
                            CREATE INDEX idx_salary_log ON employees(LOG(salary));
  6. Invisible Index
    • An index that is not considered by the optimizer unless explicitly specified in a query. 
                              
                            CREATE INDEX idx_invisible ON employees(last_name) INVISIBLE;
  7. Rebuilding and Maintaining Indexes
    • Indexes can be rebuilt or reorganized to improve performance. Use the ALTER INDEX statement for maintenance. 
                              
                            ALTER INDEX idx_last_name REBUILD;
  8. Dropping an Index
    • Use the DROP INDEX statement to remove an index from the database. 
                              
                            DROP INDEX idx_last_name;
  9. Monitoring Index Usage
    • You can query the V$OBJECT_USAGE view to monitor the usage of an index. 
                              
                            SELECT * FROM V$OBJECT_USAGE WHERE OBJECT_NAME = 'IDX_LAST_NAME';
